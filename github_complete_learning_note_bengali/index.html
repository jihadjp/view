Git Rebase — কখন ব্যবহার করবে?

Rebase মানে তোমার branch-এর commits-গুলো অন্য branch-এর উপরে "পুনরায় স্থাপন করা"। এতে commit history সুন্দর ও linear থাকে।

Merge (ugly):
A→B→C
+
D→E
→
Merge commit M
Rebase (clean):
A→B→C→D'→E'
✓
Clean history
# Interactive Rebase — শেষ 3টি commit ঠিক করো git rebase -i HEAD~3 # main-এর সাথে sync করো (merge commit ছাড়া) git checkout feature/my-feature git rebase main # Conflict হলে: git add . # conflict ঠিক করে add করো git rebase --continue # rebase চালু রাখো git rebase --abort # rebase বাতিল করো
⚠️ Golden Rule: Shared/public branch-এ কখনো Rebase করো না। শুধু তোমার নিজের local feature branch-এ rebase করো।

Git Cherry-pick — কখন ব্যবহার করবে?

অন্য branch থেকে শুধু একটি নির্দিষ্ট commit তোমার branch-এ নিয়ে আসতে চাইলে।

# Use case: hotfix-এর commit টা main-এও দরকার git log --oneline # commit hash দেখো # Output: abc1234 Fix critical payment bug git checkout main git cherry-pick abc1234 # শুধু ওই commit নাও # একাধিক commit: git cherry-pick abc1234 def5678
✅ Cherry-pick কখন দরকার: কোনো feature branch-এ একটি bug fix করা হয়েছে যেটা production-এ (main) এখনই দরকার, কিন্তু পুরো feature এখনো ready না।

Git Stash — কখন ব্যবহার করবে?

অর্ধেক কাজ করা অবস্থায় হঠাৎ অন্য কাজ করতে হলে, commit না করেই কাজ "সরিয়ে রাখো"।

# সিনারিও: feature-এ কাজ করছো, boss বললো urgent bug fix করো git stash push -m "WIP: user dashboard feature" # এখন অন্য branch-এ যাও, bug fix করো git checkout main # ... bug fix করো ... # আবার ফিরে এসো git checkout feature/dashboard git stash pop # শেষ stash ফেরত আনো # সব stash দেখো git stash list # Output: # stash@{0}: WIP: user dashboard feature # stash@{1}: Old unfinished work git stash apply stash@{1} # নির্দিষ্ট stash apply করো

GitHub Actions হলো GitHub-এর built-in CI/CD platform। Code push করলে automatically test চালাবে, build করবে, deploy করবে।

Code Push
→
Trigger Workflow
→
Run Tests
→
Build Docker
→
Deploy 🚀

বাস্তব উদাহরণ: Python App — Test → Build → Deploy

ফাইলের অবস্থান: .github/workflows/ci-cd.yml

# .github/workflows/ci-cd.yml name: CI/CD Pipeline on: push: branches: [main, develop] pull_request: branches: [main] jobs: test: runs-on: ubuntu-latest steps: - name: Checkout code uses: actions/checkout@v4 - name: Setup Python 3.11 uses: actions/setup-python@v4 with: python-version: '3.11' cache: 'pip' - name: Install dependencies run: | pip install -r requirements.txt pip install pytest pytest-cov - name: Run Tests with Coverage run: pytest --cov=app --cov-report=xml - name: Upload coverage report uses: codecov/codecov-action@v3 build-and-push: needs: test # test pass হলেই চলবে runs-on: ubuntu-latest if: github.ref == 'refs/heads/main' steps: - uses: actions/checkout@v4 - name: Login to DockerHub uses: docker/login-action@v3 with: username: ${{ secrets.DOCKERHUB_USERNAME }} password: ${{ secrets.DOCKERHUB_TOKEN }} - name: Build & Push Docker Image uses: docker/build-push-action@v5 with: push: true tags: | myapp/backend:latest myapp/backend:${{ github.sha }} deploy: needs: build-and-push runs-on: ubuntu-latest environment: production # manual approval লাগতে পারে steps: - name: Deploy to Server via SSH uses: appleboy/ssh-action@v1 with: host: ${{ secrets.SERVER_HOST }} username: ${{ secrets.SERVER_USER }} key: ${{ secrets.SSH_PRIVATE_KEY }} script: | cd /opt/myapp docker pull myapp/backend:latest docker-compose up -d --no-deps backend echo "✅ Deployed successfully!"
🔐 Secrets: GitHub-এ Settings → Secrets → Actions-এ sensitive data (password, API key) রাখো। কখনো সরাসরি YAML-এ লিখো না।

দরকারি Actions Patterns

actions/checkout@v4
কোড checkout করে
actions/cache@v3
dependencies cache করে (build দ্রুত করে)
actions/upload-artifact
build output সংরক্ষণ করে
github.sha
current commit-এর unique hash

দিন ১ — ভিত্তি গড়ো (Foundation)
সকাল (২ ঘন্টা): Git ইনস্টল, config সেট, SSH key তৈরি করো। GitHub অ্যাকাউন্ট খোলো।
দুপুর (২ ঘন্টা): প্রথম repo তৈরি করো। add, commit, push, pull, clone — এই ৫টি কমান্ড আয়ত্ত করো।
বিকাল (২ ঘন্টা): Branch তৈরি করো, feature branch থেকে Pull Request দাও এবং merge করো।
রাতে (১ ঘন্টা): Profile README তৈরি করো। GitHub Flow অনুসরণ করে একটি ছোট প্রজেক্ট তৈরি করো।
🎯 লক্ষ্য: GitHub-এ প্রথম সবুজ contribution square!
দিন ২ — Advanced Git (গভীরে যাও)
সকাল (২ ঘন্টা): Merge vs Rebase পার্থক্য হাতে-কলমে করো। Interactive rebase দিয়ে commit message ঠিক করো।
দুপুর (২ ঘন্টা): Merge conflict তৈরি করো এবং manually সমাধান করো। Cherry-pick ব্যবহার করে দেখো।
বিকাল (২ ঘন্টা): Git Stash ব্যবহার করো। git log --graph --all দিয়ে history visualization করো।
রাতে (১ ঘন্টা): Git Flow অনুসরণ করে একটি প্রজেক্টে develop, feature, release branch তৈরি করো।
🎯 লক্ষ্য: যেকোনো Git conflict সমাধান করতে পারা।
দিন ৩ — Automation & প্রোফাইল (Pro Level)
সকাল (২ ঘন্টা): GitHub Actions শেখো। একটি simple workflow তৈরি করো যেটা push হলে echo "Hello CI!" চালায়।
দুপুর (৩ ঘন্টা): Python/Node project-এ real CI pipeline তৈরি করো — lint → test → build। Secrets ব্যবহার করো।
বিকাল (২ ঘন্টা): একটি open source project-এ "good first issue" খুঁজে বের করো এবং সমাধান করার চেষ্টা করো।
রাতে (১ ঘন্টা): GitHub প্রোফাইল polish করো। ৩টি সেরা project pin করো। Tech topics যোগ করো।
